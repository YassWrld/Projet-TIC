Chapitre 1

Les techniques de l’information et de la communication

Les TIC englobent les outils pour la gestion de l'information, la production, la circulation, la transmission, la modification et l'échange de données. Elles permettent la diffusion et le stockage de l'information, ainsi que la présentation dans divers formats. Les TIC sont nées de la convergence de l'informatique, des télécommunications, de l'électronique et de l'audiovisuel.
TIC regroupent les outils (matériels et logiciels) pour communiquer, transmettre, accéder aux informations, stocker, manipuler et produire (texte, image, son, vidéo) en utilisant l'informatique, l'audiovisuel, le multimédia et les télécommunications. Les outils incluent les réseaux, les navigateurs, les moyens de stockage, les PC, tablettes, scanner, imprimante, logiciels de production, etc.

Catégories
  1. Téléinformatique : Internet, serveurs, réseaux
  2. Télécommunication : Signal, transmission, téléphone mobile
  3. Électronique : Composants, équipement grand public
Objectifs
  1. Connectivité
  2. Modularité
  3. Implantation simple
  4. Utilisation facile
  5. Fiabilité
Composants de TIC

  • La partie matérielle :

    Pcs portables, télévisions, téléphones portables, tablettes interactives, appareils photos numériques, robots, stations de travail (PC, MAC, SUN Terminal X, ...), serveurs...

  • La partie logicielle :

    Systèmes d’exploitation, logiciels d’application et d’utilisation des différents composants matériels, les réseaux d’internet ou mobiles
Caractéristiques
  • Rapidité dans la réalisation des activités
  • Décentralisation du pouvoir
  • Meilleur accès aux informations pour l'ensemble du personnel
  • Rationalité pour le partage d'informations confidentielles
  • Mondialisation de l'activité
  • Répartition de l'information entre un grand nombre de stations avec une qualité excellente
  • Intensité de la transmission et réception d'informations, en tout temps et en tout lieu
Initiation aux techniques de communication

  • Objectifs de communication

    • Cognitif : Prise de conscience, attention, notoriété, information sur marque/produit/entreprise. Objectif cognitif. Communication informative
    • Affectif : Modifier ou renforcer l'image de produit/marque en agissant sur attitudes/opinions
    • Comportemental : Modifier comportements (achat, essai)

  • Système de communication

    • Partenaires de communication : Émetteur (transforme message en signaux) et récepteur (reçoit signaux et reconstruit message)
    • Vecteurs de communication : Message (émetteur émet message et tente de le coder parfaitement) et Médias (moyen de transmission du message)
    • Fonctions de communication : Codage (traduction message en signifiants structurés), Décodage (interprétation du récepteur), Réponse (avis du client), Feedback (réaction du récepteur) Feedback peut être immédiat (communication personnalisée) ou retardé (communication de masse)
    • Bruit : Peut perturber les signaux pendant la transmission

  • Types de communication

    • Communication institutionnelle (Corporate) : Communiquer sur l'image de marque de l'entreprise/institution. Réalisée par publicité, mécénat, parrainage, sponsoring, etc.
    • Communication commerciale/produit : Mettre en avant caractéristiques, performances, avantages du produit de manière crédible et cohérente

  • Les schémas de communication

    • Communication interpersonnelle : Utilisée pour s'adresser à un collaborateur. Émetteur transmet message(s) à destinataire. Feedback possible
    • Communication de groupe : Utilisée pour s'adresser à des salariés, une équipe, ou lors de réunions. Émetteur veut faire passer un message à plusieurs destinataires. Feedback/discussion possible pour tirer le maximum d'avis de chacun
    • Communication de masse : Exemple de publicité. Feedback généralement pas attendu. Efficacité mesurée en termes de ventes ou autres

  • Média et Multimédia

    • Média : Moyen impersonnel de diffusion large et collective d'informations. Supports : presse, édition, affichage publicitaire, radio, télévision, cinéma, internet
    • Multimédia : Ensemble techniques/produits permettant utilisation simultanée/interactive de plusieurs modes représentation de l'information (textes, sons, images fixes/animées)

Chapitre 2

Réseau (informatique) Network
  • Réseau informatique de périphériques connectés géographiquement pour échanger des informations numériques selon des règles définies
  • Networking : Relier des ordinateurs pour partager des ressources via des outils et des tâches
  • Un réseau relie des équipements pour communiquer des données entre applications, peu importe la distance
  • Évolution des réseaux informatiques : Originairement pour les transferts de données, mais aujourd'hui pour la voix, la vidéo et les données. Utilisés pour connecter des terminaux distants à un site central, puis pour interconnecter des ordinateurs entre eux
  • Un réseau repose sur deux concepts clés : l'interconnexion pour la transmission de données d'un nœud à un autre et la communication pour l'échange de données entre processus. Les extrémités d'une connexion sont appelées nœuds (nodes) et les programmes en cours d'exécution qui communiquent sur le réseau sont des processus
Les caractéristiques de base d'un réseau
  • Topologie des réseaux : La topologie physique décrit l'interconnexion des équipements, tandis que la topologie logique décrit la manière dont les équipements communiquent entre eux
  • Débit : Quantité de données numériques transmises par seconde (bps ou bits/s)
  • Portée maximale : Dépend de la technologie utilisée
  • Nombre maximum de nœuds interconnectables
Quelques terminologies en réseau
  • Station de travail : Machine capable d'envoyer des données sur des réseaux (PC, MAC, SUN Terminal X, etc.)
  • Nœud : Station de travail, imprimante, serveur ou toute autre entité adressable par un numéro unique
  • Serveur : Entité dépositaire d'une fonction spécifique telle que la base de données, le traitement, les fichiers, le courrier électronique, etc.
  • Paquet : Plus petite unité d'information pouvant être envoyée sur un réseau. Contient généralement l'adresse de l'émetteur, celle du destinataire et les données à transmettre
  • Réseaux homogènes : Tous les ordinateurs sont du même constructeur, comme AppleTalk utilisé par les ordinateurs Apple Macintosh
  • Réseaux hétérogènes : Les ordinateurs connectés au réseau sont de fabricants différents, tels qu’Ethernet qui utilise un câble coaxial
Éléments d'un réseau
  • Ordinateurs équipés d'une carte de communication
  • Logiciels de navigateur, client de messagerie, serveur web, etc.
  • Supports du LAN : câbles paires cuivre torsadées, prises RJ45, WIFI, etc.
  • Supports du WAN : ligne téléphonique, ADSL, fibre optique, etc.
  • Équipements d'interconnexion du LAN : répéteur (transceiver), concentrateur (hub), commutateur (switch), etc.
  • Équipements d'interconnexion du WAN : routeur, etc.
Classification des réseaux

  1. Par portée l'envergure (La distance)

    • LAN : Réseau local d'entreprise constitué d'ordinateurs et de périphériques reliés entre eux, qui ne dépasse pas généralement la centaine de machines et qui a une portée de quelques centaines de mètres à quelques kilomètres. Les vitesses de transmission vont de 10 à 100 Mb/s
    • MAN : Réseau qui regroupe plusieurs réseaux locaux (LAN) dans une même zone géographique, comme une grande entreprise ou une ville. Sa portée peut aller jusqu'à 10 à 25 km, et le câble co-axial est souvent utilisé comme support physique
    • WAN : Réseau étendu qui assure la transmission de données sur de longues distances à l'échelle d'un pays ou de la planète. Ce type de réseau est généralement public et géré par des opérateurs publics ou privés

  2. L'ouverture (leur utilisation)

    Les réseaux peuvent être classés en fonction des services qu'ils offrent. Les réseaux TCP/IP sont classés en trois types : l'Intranet, réservé aux entités organisationnelles ou aux entreprises ; l'Extranet, ouvert vers l'extérieur ; et Internet, le réseau public national ou international interconnecté à l'échelle de la planète.

  3. Les supports de connexion : filaires et non filaires

    • Réseaux filaires : Utilisent des câbles comme le coaxial, les paires torsadées, la fibre optique, les lignes téléphoniques
    • Réseaux sans fil : Utilisent des moyens comme les ondes radios, les ondes infrarouges, les satellites, le Bluetooth

  4. Le type d'organisation : Peer to Peer et client serveur

    • Réseaux Postes à Postes (Peer to Peer) : Chaque ordinateur est à la fois client et serveur, libre de partager ses ressources
    • Réseaux Clients/Serveurs : La plupart des stations sont des postes clients, tandis que d'autres stations sont dédiées à des tâches spécialisées en tant que serveurs
Le type de topologie
  • Réseaux en bus : Les machines sont reliées par un câble coaxial en série sur le bus, avec une limitation de débit à 10 Mbits/s
  • Réseaux en étoile : Toutes les stations sont connectées à un commutateur ou un nœud central
  • Réseaux en anneau : Les ordinateurs sont connectés en boucle et communiquent chacun à leur tour en utilisant la méthode d'accès à "jeton"
  • Réseaux maillés : Utilisent plusieurs chemins de transfert entre les différents nœuds pour garantir la stabilité en cas de panne

Les Composants matériel d'un Réseau informatique

Les cartes réseau

aussi appelées cartes d'interface réseau , sont des composants matérielsqui permettent à un ordinateur de se connecter à un réseau, comme Internet ou un réseau local.Les cartes réseau sont installées à l'intérieur de l'ordinateur et sont généralement connectées àune prise Ethernet ou Wi-Fi.Les cartes réseau jouent un rôle essentiel dans la communication entre ordinateurs sur unréseau. Elles permettent à l'ordinateur de transmettre et de recevoir des données via desprotocoles de communication spécifiques, tels que TCP/IP, qui sont utilisés pour transférer desdonnées sur Internet.

SWITCH

Un switch est un composant matériel utilisé pour connecter des appareils sur unréseau local . Le switch permet à plusieurs appareils, tels que des ordinateurs, des imprimantes,des serveurs, etc., de communiquer entre eux en utilisant des adresses MAC .Lorsqu'un appareil se connecte au switch, le switch vérifie son adresse MAC et stocke cetteinformation dans une table de correspondance. Lorsqu'un autre appareil sur le réseau souhaitecommuniquer avec cet appareil, le switch utilise cette table pour envoyer les donnéesuniquement à l'appareil destinataire, plutôt que de les diffuser à tous les appareils du réseau.Les switches sont disponibles en différentes tailles, allant des petits switches pour les réseauxdomestiques et les petits bureaux, aux switches de niveau entreprise pour les réseaux plusimportants.

OSI

est un modèle de référence pour les communications de réseau qui divise lescommunications en sept couches distinctes. Chaque couche est responsable de fonctionsspécifiques liées à la communication de données entre les appareils de réseau.Le modèle OSI est souvent utilisé comme référence pour la conception et la mise en œuvre deréseaux informatiques.

Introduction

Le web et internet c'est pareil : Le WWW (WORLD WIDE WEB), littéralement la « toile mondiale ». Le Web permet de consulter, avec un navigateur, des pages accessibles sur des des sites. L'image de la toile d'araignée vient des hyperliens qui relient les pages web entre elles.

Aspects techniques du Web
Les 3 mécanismes de base du Web sont :

  1. URL
    1. Le Web permet d'accéder à un ensemble de ressources
    2. Le mécanisme de localisation peut faire appel au protocole DNS
  2. HTTP
    1. Protocole de niveau applicatif
    2. Paradigme client-serveur
    3. Protocole sans état (pas de « mémoire » des transactions précédentes)
  3. HTML
    1. Langage de description de « pages Web »
    2. Texte, images et autres objets
    3. Liens hypermédias entre les pages
    4. Programmation déclarative
Page web et site web

Une page Web, c'est un document multimédia : images, textes, possibilité d'interaction, liens ...etc. Décrit élément par élément : titre, morceaux de texte, images...etc.
Avec un langage de description :HTML (1992), XHTML (2002) ou HTML5

  • Stocké dans un fichier : html (ou .htm, .xhtml)
  • Une page Web est interprétée et affichée par un navigateur, elle est localisée sur Internet à l'aide d'une adresse (URL)
  • Permet d'accéder à d'autres pages en suivant des liens
  • Un site web est un regroupement de pages Web autour d'une thématique commune, reliés entre elles par des liens hypertextes
  • Émanant d'une même entité (organisation, entreprise, particulier...), accessible à partir d'une page d'accueil commune et à partir d'une URL de base commune
  • En général, localisées sur un même serveur
  • Structuré sous forme d'arborescence de dossiers et de fichiers (comme un disque local)
  • Permet de définir des chemins relatifs
Création d'une page web HTML

HTML : HyperText Markup Language Langage de description de pages web
Un document HTML est :

  • Un fichier texte, intégré suivant une syntaxe précise
  • Structuré par des balises
  • Interprété (visualisé) dans un navigateur (Mozilla Firefox, Internet Explorer, Safari, etc.) USTHB-FI TIC Page 4 / 4

Chapitre 4

Un système de gestion de version est un outil logiciel de visioning pour gérer les versions de fichiers. Il permet de :

  • Suivre les modifications
  • Facilite le travail en équipe
  • Fusionne les contributions et permet de revenir à des versions précédentes.

Les systèmes peuvent être centralisés ou décentralisés, tels que Git, Subversion et Mercurial

GIT & GitHub

Créé en 2005 par Linus Torvalds, c’est un logiciel de visioning décentralisé le plus utilisé dans le développement de logiciels. Il permet de suivre les modifications apportées aux fichiers, de coordonner le travail d'équipe , et de conserver un historique complet de modifications . Git offre une grande flexibilité et une gestion efficace des branches, ce qui permet de travailler simultanément sur différentes versions d'un même projet. Il facilite la gestion de tâches tout en étant rapide , il propose également des services d'hébergement , essentiellement GitHub. C'est un outil indispensable pour les développeurs souhaitant gérer leurs projets et collaborer de manière à être à jour et productif. ➔ Les branches nous permettent d’avoir un timeline parallèle dans lequel on va pouvoir faire des essais et voir l’avancement du projet et amener des modifications .

Les deux modèles des logiciels de gestion de version modèle centralisé vs modèle décentralisé:

  • modele centralisé :

    Le code source est hébergé sur un serveur distant et les utilisateurs doivent se connecter à ce serveur pour travailler dessus. Toutes les données sont donc sur le serveur central , ce qui est un inconvénient si le serveur se met en panne tout sera perdu. Il est aussi relativement lent pour publier les grosses modifications .

  • modele décentralisé :

    Chaque utilisateur télécharge et héberge l'intégralité du code source sur sa propre machine et va avoir un historique du projet , les modifications se font localement et une fois satisfait , le développeur envoie une copie au serveur central . L’avantage c'est que c'est distribué donc ça permet d’aller plus vite et d’avoir des données plus sécurisées vu qu’elles sont duplique .
Utiliser GIT :

  • Installation de Git

    l’installation de Git se fait à partir du site officiel http://git-scm.com/downloads. Une fois le fichier téléchargé, vous devez l'ouvrir et suivre les instructions en laissant les valeurs par défaut

  • Paramétrage de Git

    Paramétrer Git en renseignant un nom d'utilisateur et une adresse email est important pour ne pas avoir à les fournir à chaque utilisation. La commande git config permet de modifier les variables de configuration en et ajouter --global permet de les utiliser globalement. Pour le faire , on tape git config --global user.name 'nom et git config --global user.email 'adresse email. on tape "git config user.name" et "git config user.email", pour vérifier si elles ont bien été enregistrer .

  • Démarrer un dépôt Git

    Un dépôt Git est une copie complète des fichiers d'un projet , Deux façons de le créer : importer un répertoire existant ou cloner un dépôt Git existant.

  • La gestion des informations selon Git

    Git considère les données comme un flux d'instantanés ou "snapshots". À chaque fois que l'on enregistre l'état d'un projet, Git prend un instantané du contenu de l'espace de travail et enregistre une référence à cet instantané. Chaque instantané est stocké localement dans une base de données. Cela permet de travailler sur l'historique complet d'un projet localement, ce qui rend les opérations plus rapides et plus agréables.

  • Les états des fichiers

    Git suit les modifications sur les fichiers d'un projet en les considérant comme des instantanés. Un fichier peut être suivi ou non suivi, et s'il est suivi, il peut être dans l'un des trois états : modifié, indexé ou validé. Lorsqu'on démarre un dépôt Git, les fichiers sont non suivis et on doit les indexer et les valider. Lorsqu'on clone un dépôt existant, les fichiers sont déjà suivis et validés. Les fichiers modifiés sont considérés comme modifiés par Git tant qu'ils n'ont pas été indexés. Lorsqu'on demande à Git de prendre un instantané, les fichiers indexés et non modifiés sont validés et le cycle peut recommencer.

  • Les zones de travail

    Un projet Git est composé de trois sections : le répertoire de travail, la zone d’index et le répertoire Git. Les fichiers peuvent être dans l'état modifié, indexé ou validé. Le répertoire est l'endroit où l'on travaille sur les fichiers, la zone d'index est le fichier qui stocke les informations qui feront partie du prochain instantané , et le répertoire stocke les métadonnées et la base de données du projet. On travaille sur les fichiers dans le répertoire de travail, on les indexe pour qu'ils soient prêts pour le prochain instantané, puis on les valide pour l' ajouter au répertoire Git.

  • Créer un dépôt Git à partir d’un répertoire existant

    On utilise la commande git init pour commencer la création d’un dépôt . La commande git status affiche l'état des fichiers de notre répertoire. Pour indexer des fichiers, on utilise la commande git add, puis pour les valider et les ajouter en base, on utilise la commande git commit.

  • Cloner un dépôt Git

    La deuxième façon de démarrer un dépôt Git est de cloner un dépôt Git existant en utilisant la commande Git clone. Transmettez les fichiers à un repo GitHub après l’avoir créé .

  • Ajouter ou modifier des fichiers dans un projet et actualiser notre dépôt Git

    Pour ajouter, modifier ou supprimer des fichiers,on utilise les commandes git add et git commit. Le commit d'un fichier est basé sur l'état du fichier au moment du dernier git add, Pour éviter que les dernières modifications ne se perdent , il est recommandé d’ effectuer un git add juste avant un git commit .

  • Consulter l’historique des modifications Git

    La commande git log affiche l'historique des modifications Git, qui présente la liste des commits du plus récent au plus ancien avec leur somme de contrôle, le nom de l'auteur, la date et le message du commit.

  • Annuler des modifications apportées à un fichier

    On peut utiliser un logiciel de gestion de version pour revenir à un état antérieur d'un projet. Pour cela on utilise la commande git checkout - nom-du-fichier ou git restore.

  • Les branches GIT ( ou master )

    Les branches sont une copie du projet principale pour tester des fonctionnalités ou les modifier , sans impacter le projet de base . Contrairement au autre système de gestion , une branche (master) est un pointeur vers un commit spécifique dans Git , c'est fait pour éviter de recopier l'intégralité du répertoire de travail et de rendre l'opération de création d’une branche plus légère .
